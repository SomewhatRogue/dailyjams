Interaction Flow & Navigation

1
Trigger: User clicks "Quick Start" button in nav (or automatically on first visit)
2
Overlay Appears: Backdrop fades in (0.4s), page content dims, pointer-events blocked except on tooltip
3
Step Sequence: Spotlight animates to target element (0.4s smooth transition), tooltip enters with bounce
4
Navigation Options: "Next" advances, "Skip Tour" closes immediately, progress dots show position
5
Completion: "Let's Go!" button closes overlay, sets localStorage flag onboardingComplete: true
6
Dismissal: Clicking backdrop (outside tooltip) also closes tour, pressing ESC key closes

Animation & Timing Specifications

Button Animations

Pulse Glow: 2s ease-in-out infinite

Shine Sweep: 3s ease-in-out infinite

Hover: Disables animations, lifts -2px

Overlay Transitions

Backdrop Fade: 0.4s ease

Spotlight Move: 0.4s ease

Tooltip Enter: 0.4s cubic-bezier bounce

Spotlight Effect

Box Shadow: 0 0 0 9999px for full-screen dim

Border: 3px solid gold with pulsing glow

Pulse: 2s ease-in-out infinite

Tooltip Positioning

Auto-detect: Calculates best position (above/below/left/right)

Arrow: Rotated pseudo-element with border

Max-width: 360px for readability

Recommended Walkthrough Steps

These are the 5 key areas that should be explained in the onboarding tour:

1. Vibe Input (Main Form)

The primary text field where users describe their mood. Explain this is the core of discovery.

2. Discovery Arc Slider

The sun-to-moon arc that controls familiar vs. new. Show how it affects recommendations.

3. Swipe Cards (Discover Page)

Explain swipe gestures: right to add, left to pass, down for more options.

4. Build Playlist Button

Show the floating button that creates Spotify playlists from saved discoveries.

5. Completion / Celebration

End with encouragement. User is ready to discover music.

Accessibility & UX Considerations

Keyboard Navigation: Tab cycles through tooltip buttons. ESC closes overlay. Enter/Space trigger buttons.
Focus Management: When overlay opens, focus moves to first tooltip button. When closed, returns to trigger button.
Screen Readers: Overlay has role="dialog", aria-modal="true", aria-labelledby pointing to tooltip title.
Reduced Motion: If prefers-reduced-motion is set, disable animations (use instant transitions).
Persistence: Store in localStorage: onboardingComplete, onboardingSkipped. Don't show again unless user clicks "Quick Start".
Mobile Adaptation: Tooltip max-width adjusts on mobile. Touch events work for dismissal. Progress dots stay visible.

Implementation Architecture

Suggested file structure and approach for building this system:

/frontend/static/css/onboarding.css
  - All styles for overlay, spotlight, tooltip
  - Animations: pulse-glow, shine, tooltip-enter

/frontend/static/js/onboarding.js
  - OnboardingManager class
  - Methods: start(), nextStep(), skipTour(), calculatePosition()
  - Event handlers: keyboard, click, escape
  - localStorage persistence

/frontend/templates/partials/onboarding.html
  - Overlay structure (backdrop, spotlight, tooltip)
  - Can be included in base template

Integration:
  1. Add button to header nav with shining styles
  2. Include onboarding.html partial
  3. Initialize OnboardingManager on page load
  4. Check localStorage - if not complete & first visit, auto-start
  5. Steps defined as JSON config (target selector, title, description, position)
